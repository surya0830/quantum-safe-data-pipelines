# Contributing to Quantum-Safe Data Pipelines

Thank you for your interest in contributing to this research project!

## License

By contributing to this project, you agree that your contributions will be licensed under the MIT License.

## Code of Conduct

This is an academic research project. We expect all contributors to:
- Be respectful and constructive in discussions
- Focus on scientific rigor and reproducibility
- Clearly document any assumptions or limitations

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest enhancements
- Include clear reproduction steps for bugs
- For security issues, please email the maintainer directly

### Contributing Code
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes with clear commit messages
4. Add tests if applicable
5. Update documentation (README, paper, docstrings)
6. Submit a pull request

### Code Standards
- Follow PEP 8 style guidelines for Python code
- Add docstrings to all public functions and classes
- Include type hints where appropriate
- Add the MIT license header to new source files (see `.license-header.py`)

### Research Standards
- Document all experimental methodology clearly
- Use fixed random seeds for reproducibility
- Update benchmark results when changing implementations
- Clearly distinguish educational implementations from production-ready code

### Areas for Contribution

We especially welcome contributions in:
- **Real PQC Integration**: Fixing remaining API compatibility issues with liboqs
- **Performance Benchmarking**: Testing on different hardware configurations
- **Spark Integration**: End-to-end pipeline benchmarks at scale
- **QKD Simulation**: More sophisticated channel models
- **Documentation**: Improving clarity and adding examples
- **Testing**: Expanding test coverage

### Questions?

Open an issue or discussion on GitHub for any questions about contributing.

## Acknowledgments

Contributors will be acknowledged in the project README and research paper (for significant contributions).
